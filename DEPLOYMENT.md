# Deploying CareFlow

CareFlow is a full-stack app, so a live deploy needs three pieces:

| Piece | What it is | Where it runs |
|---|---|---|
| **Database** | PostgreSQL | Render Postgres (or Neon) |
| **API** | Node + Express + Prisma | Render web service |
| **Client** | React + Vite static build | Render static site (or Vercel) |

Local dev keeps using **SQLite** (zero setup). The build automatically swaps Prisma to
**PostgreSQL** for the cloud, so nothing changes on your machine.

---

## Option A — One blueprint on Render (recommended, single free account)

The repo ships a [`render.yaml`](render.yaml) that provisions all three pieces at once.

1. Create a free account at **https://render.com** and connect your GitHub.
2. In Render: **New → Blueprint**, pick the `careflow-hms` repo, click **Apply**.
   Render reads `render.yaml` and creates: `careflow-db`, `careflow-api`, `careflow-client`.
3. When it prompts for **`VITE_API_URL`** on the client, you can put a placeholder for now
   (e.g. `https://careflow-api.onrender.com/api`) — you'll confirm it next.
4. Wait for **`careflow-api`** to go live. Copy its URL (looks like
   `https://careflow-api-xxxx.onrender.com`).
5. Open the **`careflow-client`** service → **Environment** → set
   `VITE_API_URL = https://<your-api-url>/api` (the URL from step 4 + `/api`).
   Then **Manual Deploy → Clear build cache & deploy** so the value is baked in.
6. Open the client URL. Log in with a demo account (below). Done. 🎉

> **Free-tier notes:** the API sleeps after ~15 min idle and cold-starts in ~30–60s
> (first request after a nap is slow — that's normal). Render's free Postgres expires
> ~30 days after creation; for a permanent DB, use **Neon** (Option B).

## Option B — Neon (persistent DB) + Render API + Vercel client

Use this if you want the database to last indefinitely.

1. **Database:** create a free project at **https://neon.tech**, copy the connection string.
2. **API:** on Render, **New → Web Service** from the repo, root dir `server`.
   - Build: `npm install && sed -i 's/provider = "sqlite"/provider = "postgresql"/' prisma/schema.prisma && npx prisma generate`
   - Start: `npx prisma db push --skip-generate --accept-data-loss && node prisma/seed.js && node src/index.js`
   - Env: `DATABASE_URL` = your Neon string · `JWT_SECRET` = any long random string · `SKIP_SEED_IF_PRESENT` = `1`
3. **Client:** on **https://vercel.com**, import the repo, root dir `client`,
   framework **Vite**. Add env `VITE_API_URL = https://<your-api>.onrender.com/api`. Deploy.

## Security notes for a public deploy

- `JWT_SECRET` is generated by Render (Option A) — never commit a real secret.
- CORS reflects the caller's origin by default. To lock it down, set `CLIENT_ORIGIN`
  on the API to your exact client URL.
- All data is **synthetic** — never load real patient information into a public demo.
